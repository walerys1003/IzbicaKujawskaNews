# Contributing

## Branching

- `main` — produkcja
- `feature/*` — nowe funkcje
- `fix/*` — poprawki
- `chore/*` — utrzymanie

## Standard zmian

- Każda zmiana powinna mieć opis biznesowy.
- Każdy nowy endpoint lub binding aktualizuje dokumentację.
- Migracje D1 muszą być idempotentne.
- Nazwy plików i tras utrzymujemy w konwencji kebab-case.

## Minimalny Definition of Done

- build przechodzi
- dokumentacja zaktualizowana
- public route lub API ma smoke / opis
