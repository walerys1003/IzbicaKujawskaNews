# Security Policy

## Zgłaszanie problemów

- wysyłaj prywatnie do właściciela repozytorium
- nie publikuj exploitów ani sekretów w publicznym issue

## Zakres

- auth i JWT
- AI endpoints i prompt injection hardening
- D1 migrations
- KV / R2 bindings
- GitHub Actions i deploy

## Dobre praktyki

- rotacja sekretów po incydencie
- least privilege dla tokenów Cloudflare
- przegląd logów po każdym deployu produkcyjnym
